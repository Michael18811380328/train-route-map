根据您的数据结构，我将设计三个SQL数据库表并分析它们之间的关系。

### 数据库表设计

#### 1. 车站表 (stations)

| 字段名 | 数据类型       | 约束               | 说明                         |
|--------|---------------|--------------------|------------------------------|
| id     | INT PRIMARY KEY | AUTO_INCREMENT     | 站点ID，唯一标识符           |
| x      | FLOAT          | NOT NULL           | 站点的x坐标                 |
| y      | FLOAT          | NOT NULL           | 站点的y坐标                 |
| name   | VARCHAR(255)   | NOT NULL, UNIQUE   | 站点名称                    |
| size   | TINYINT        | NOT NULL, CHECK(1-5)| 站点等级(1-5)               |

```sql
CREATE TABLE stations (
    id INT AUTO_INCREMENT PRIMARY KEY,
    x FLOAT NOT NULL,
    y FLOAT NOT NULL,
    name VARCHAR(255) NOT NULL UNIQUE,
    size TINYINT NOT NULL CHECK (size BETWEEN 1 AND 5)
);
```

#### 2. 路线表 (routes)

| 字段名       | 数据类型       | 约束               | 说明                         |
|--------------|---------------|--------------------|------------------------------|
| id           | INT PRIMARY KEY | AUTO_INCREMENT     | 路线ID，唯一标识符           |
| name         | VARCHAR(255)   | NOT NULL           | 路线名称                    |
| color        | VARCHAR(20)    | DEFAULT 'black'    | 路线颜色                    |
| speed        | INT            | DEFAULT 100        | 路线速度(km/h)              |
| line_number  | TINYINT        | DEFAULT 1, CHECK(1-4)| 路线个数(1-4)              |

```sql
CREATE TABLE routes (
    id INT AUTO_INCREMENT PRIMARY KEY,
    name VARCHAR(255) NOT NULL,
    color VARCHAR(20) DEFAULT 'black',
    speed INT DEFAULT 100,
    line_number TINYINT DEFAULT 1 CHECK (line_number BETWEEN 1 AND 4)
);
```

#### 3. 路线站点关联表 (route_stations)

| 字段名      | 数据类型 | 约束                          | 说明                     |
|-------------|----------|-------------------------------|--------------------------|
| route_id    | INT      | FOREIGN KEY REFERENCES routes(id) | 路线ID                  |
| station_id  | INT      | FOREIGN KEY REFERENCES stations(id) | 站点ID                  |
| order_index | INT      | NOT NULL                      | 站点在路线中的顺序       |

```sql
CREATE TABLE route_stations (
    route_id INT,
    station_id INT,
    order_index INT NOT NULL,
    PRIMARY KEY (route_id, station_id),
    FOREIGN KEY (route_id) REFERENCES routes(id) ON DELETE CASCADE,
    FOREIGN KEY (station_id) REFERENCES stations(id) ON DELETE CASCADE
);
```

#### 4. 车次表 (trains)

| 字段名        | 数据类型       | 约束                         | 说明                         |
|---------------|---------------|------------------------------|------------------------------|
| id            | INT PRIMARY KEY | AUTO_INCREMENT              | 火车ID，唯一标识符           |
| train_number  | VARCHAR(50)    | NOT NULL, UNIQUE            | 火车编号，唯一               |
| start         | INT            | FOREIGN KEY REFERENCES stations(id) | 起点站ID             |
| end           | INT            | FOREIGN KEY REFERENCES stations(id) | 终点站ID             |
| color         | VARCHAR(20)    | DEFAULT 'black'             | 火车颜色                    |
| route_id      | INT            | FOREIGN KEY REFERENCES routes(id) | 所属路线ID           |

```sql
CREATE TABLE trains (
    id INT AUTO_INCREMENT PRIMARY KEY,
    train_number VARCHAR(50) NOT NULL UNIQUE,
    start INT,
    end INT,
    color VARCHAR(20) DEFAULT 'black',
    route_id INT,
    FOREIGN KEY (start) REFERENCES stations(id) ON DELETE SET NULL,
    FOREIGN KEY (end) REFERENCES stations(id) ON DELETE SET NULL,
    FOREIGN KEY (route_id) REFERENCES routes(id) ON DELETE SET NULL
);
```

#### 5. 车次时刻表 (train_times)

| 字段名     | 数据类型 | 约束                          | 说明                     |
|------------|----------|-------------------------------|--------------------------|
| train_id   | INT      | FOREIGN KEY REFERENCES trains(id) | 车次ID                  |
| station_id | INT      | FOREIGN KEY REFERENCES stations(id) | 站点ID                  |
| time       | INT      | NOT NULL                      | 到达时间(分钟)           |
| order_index| INT      | NOT NULL                      | 在车次时刻表中的顺序     |

```sql
CREATE TABLE train_times (
    train_id INT,
    station_id INT,
    time INT NOT NULL,
    order_index INT NOT NULL,
    PRIMARY KEY (train_id, station_id),
    FOREIGN KEY (train_id) REFERENCES trains(id) ON DELETE CASCADE,
    FOREIGN KEY (station_id) REFERENCES stations(id) ON DELETE CASCADE
);
```

### 表关系分析

1. 一对多关系:
   - 一个路线包含多个站点 (routes ↔ route_stations)
   - 一个车次属于一个路线 (trains.route_id → routes.id)
   - 一个车次有多个时刻记录 (trains ↔ train_times)

2. 多对多关系:
   - 路线与站点之间是多对多关系，通过route_stations表实现
   - 车次与站点之间是多对多关系，通过train_times表实现

3. 关联关系:
   - `route_stations` 表连接 routes 和 stations
   - `train_times` 表连接 trains 和 stations
   - `trains` 表通过外键直接关联到 routes 和 stations

### 关键关系说明

1. 路线-站点关系:
   - 每条路线由多个站点组成，存储在route_stations表中
   - order_index字段确保站点在路线中的正确顺序

2. 车次-路线关系:
   - 每个车次属于一个路线（通过trains.route_id）
   - 车次继承路线的站点顺序（通过route_stations）

3. 车次-时刻表关系:
   - 每个车次在每个站点有特定的到达时间
   - time字段存储从0:00开始的分钟数（如480表示8:00）
   - order_index确保时刻表顺序与路线站点顺序一致

4. 起点/终点关系:
   - 车次的起点(start)和终点(end)引用stations表
   - 这些站点必须包含在车次所属路线的站点中

### 数据完整性保障

1. 外键约束：确保所有关联数据的一致性
2. 级联删除：
   - 删除路线时，自动删除关联的route_stations记录
   - 删除车次时，自动删除关联的train_times记录
3. 唯一约束：
   - 站点名称唯一
   - 车次编号唯一
4. 检查约束：
   - 站点等级(1-5)
   - 路线数量(1-4)

这种设计提供了良好的数据完整性和灵活性，支持您火车运行模拟器的需求，同时优化了查询效率。
